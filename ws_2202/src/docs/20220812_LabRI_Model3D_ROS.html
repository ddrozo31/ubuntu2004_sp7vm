Definición de componentes (URDF)¶

  • Definicin URDF para un cuerpo rgido.

    <link name="<link_name>">
  <visual>
      <!-- ... -->
  </visual>

  <inertial>
      <!-- ... -->
  </inertial>

  <collision>
      <!-- ... -->
  </collision>
</link>

  • Parmetros visuales y estuctura de colisiones.

    <geometry>
  <box size="<x0 y0 z0>"/>
</geometry>
<origin xyz="<x0 y0 z0>" rpy="<r0 p0 y0>"/>

    <geometry>
  <cylinder radius="<d0>" length="<l0>"/>
</geometry>
<origin xyz="<x0 y0 z0>" rpy="<r0 p0 y0>"/>

    <geometry>
  <mesh filename="package://<pkg/meshes/file_name.stl>" scale="<s0 s1 s2>"/>
</geometry>
<origin xyz="<x0 y0 z0>" rpy="<r0 p0 y0>"/>
  • Parmetros inerciales.

Estructura general de una Articulación¶

  • Entre 2 cuerpos rgidos siempre debe definirse una articulacin. Existen diferentes types: revolute, continuous, prismatic, fixed, floating, and planar.
    <joint name="<joint_tag>" type="<joint_type>">
  <parent link="<link_tag1>"/>
  <child link="<link_tag2>"/>
  <origin xyz="<x0 y0 z0>" rpy="<r0 p0 y0>"/>
  <axis xyz="<i j k>"/>
  <limit effort="<e0>" lower="<l0>" upper="<u0>" velocity="<v0>"/>
  <dynamics damping="<d0>" friction="<f0>"/>  
</joint>

Definición de Propiedades y Macros¶

  • Una propiedad es un valor o variable que puede ser utilizado en todo el documento.

    <xacro:property name="<var_name>" value="<p0>" />

  • Para utilizar la propiedad en una definicin se debe emplear "${<var_name>}".

    <link name="<link_tag>">
      <!-- ... -->
           <box size="${<var_name>}"/>
      <!-- ... -->
  </link>

  • Una macro es una estructura rgida con parmetros que permite encapsular definiciones.

    <xacro:macro name="<macro_name>" params="<var_1 var_2 ... var_n>">
  <!-- ... -->
</xacro:macro>
  • Para utilizar la macro se inicia por el nombre de la misma y se definen todos su parmetros.
    <xacro:macro_name var_1 = "v0" var_2 = "v1" "..." var_n="vn" />

Ejecutar Modelo Robot¶

  • Crear un carpeta llamada launch en la raz del paquete modelrobot_pkg_gYY y agregar un archivo con nombre robot_launch_gYY.launch.py.

  • Un archivo Launch permite encapsular informacin de nodos y parmetros de tal forma que puedan ser ejecutados de forma secuencial a partir de un nico ejecutable.

import os

from ament_index_python.packages import get_package_share_path

from launch import LaunchDescription
from launch_ros.actions import Node

from launch.substitutions import LaunchConfiguration
from launch.actions import DeclareLaunchArgument
from launch.conditions import IfCondition, UnlessCondition

import xacro

pkg_folder = 'modelrobot_pkg_g00'
robot_file = 'robot_car_g00.urdf.xacro'

def generate_launch_description():
    pkg_path = os.path.join(get_package_share_path(pkg_folder))
    default_model_path = os.path.join(pkg_path + '/model/' + robot_file)

    gui_arg = DeclareLaunchArgument(name='gui', default_value='true', choices=['true', 'false'], description='Flag to enable joint_state_publisher_gui')

    # Process the URDF file
    robot_description_config = xacro.process_file(default_model_path)

    params = {'robot_description': robot_description_config.toxml()}

    robot_state_publisher_node = Node(
        package='robot_state_publisher',
        executable='robot_state_publisher',
        output='screen',
        parameters=[params]
    )

    # Depending on gui parameter, either launch joint_state_publisher or joint_state_publisher_gui
    joint_state_publisher_node = Node(
        package='joint_state_publisher',
        executable='joint_state_publisher',
        condition=UnlessCondition(LaunchConfiguration('gui'))
    )

    joint_state_publisher_gui_node = Node(
        package='joint_state_publisher_gui',
        executable='joint_state_publisher_gui',
        name="joint_state_publisher_gui",
        condition=IfCondition(LaunchConfiguration('gui'))
    )

    # Launch!
    return LaunchDescription([
        gui_arg,
        joint_state_publisher_node,
        joint_state_publisher_gui_node,
        robot_state_publisher_node,
    ])

Ejecutar RViz desde un archivo Launch con parámetros¶

  • Crear un carpeta llamada launch en la raz del paquete modelrobot_pkg_gYY y agregar un archivo con nombre simulator_gYY.launch.py.
import os

from ament_index_python.packages import get_package_share_path

from launch import LaunchDescription
from launch_ros.actions import Node

from launch.substitutions import LaunchConfiguration
from launch.actions import DeclareLaunchArgument


pkg_folder = 'modelrobot_pkg_g00'
rviz_file = 'rviz_config.rviz'

def generate_launch_description():
    pkg_path = os.path.join(get_package_share_path(pkg_folder))
    default_rviz_config_path = os.path.join(pkg_path + '/model/' + rviz_file)

    rviz_arg = DeclareLaunchArgument(name='rvizconfig', default_value=str(default_rviz_config_path), description='Absolute path to rviz config file')


    # Depending on gui parameter, either launch joint_state_publisher or joint_state_publisher_gui
    rviz_node = Node(
        package='rviz2',
        executable='rviz2',
        name='rviz2',
        output='screen',
        arguments=['-d', LaunchConfiguration('rvizconfig')],
    )

    # Launch!
    return LaunchDescription([
        rviz_arg,
        rviz_node,
    ])
